# Phase 03.5: UI Implementation for Documents and Borrowers (GET Operations)

**Date:** 2026-05-08
**Agent:** Claude Code
**Goal:** Implement React UI pages for Documents and Borrowers with GET API integration

## Overview

This phase implements the frontend UI for viewing and managing Documents and Borrowers, integrating with the existing GET API endpoints. The implementation follows Material UI design patterns and uses Zustand for state management.

## Components Implemented

### State Management Stores

#### 1. `apps/web/src/store/documentStore.ts`

Zustand store for document state management.

**State:**
- `documents: DocumentRecord[]` - List of documents
- `selectedDocument: DocumentRecord | null` - Currently selected document
- `pagination: PaginationInfo | null` - Pagination metadata
- `filters: DocumentFilters` - Active filters (status, borrowerId)
- `isLoading: boolean` - Loading state
- `error: string | null` - Error message

**Actions:**
- `fetchDocuments(filters?, limit?, offset?)` - Fetch paginated documents
- `fetchDocumentById(id)` - Fetch single document
- `setFilters(filters)` - Update filters and refetch
- `resetFilters()` - Clear all filters
- `clearError()` - Clear error message

**Key Features:**
- Builds query parameters for API calls
- Converts ISO date strings to Date objects
- Server-side pagination support
- Status and borrowerId filtering

#### 2. `apps/web/src/store/borrowerStore.ts`

Zustand store for borrower state management.

**State:**
- `borrowers: BorrowerRecord[]` - List of borrowers
- `selectedBorrower: BorrowerRecord | null` - Currently selected borrower
- `pagination: PaginationInfo | null` - Pagination metadata
- `searchQuery: string` - Current search query
- `isLoading: boolean` - Loading state
- `error: string | null` - Error message

**Actions:**
- `fetchBorrowers(searchQuery?, limit?, offset?)` - Fetch paginated borrowers
- `fetchBorrowerById(id)` - Fetch single borrower
- `setSearchQuery(query)` - Update search query (caller should debounce)
- `clearSearch()` - Clear search and refetch all
- `clearError()` - Clear error message

**Key Features:**
- Complex date conversion handling for nested ExtractedFields
- Search functionality support
- Converts `extractedAt` dates in all ExtractedField properties

## Technical Patterns

### Date Handling

API returns ISO 8601 date strings which need to be converted to JavaScript Date objects for proper formatting in the UI.

**Simple conversion (Documents):**
```typescript
const documents = data.data.map((doc: any) => ({
  ...doc,
  uploadedAt: new Date(doc.uploadedAt),
  updatedAt: new Date(doc.updatedAt)
}));
```

**Complex nested conversion (Borrowers):**
```typescript
fullName: borrower.fullName ? {
  ...borrower.fullName,
  extractedAt: borrower.fullName.extractedAt
    ? new Date(borrower.fullName.extractedAt)
    : undefined
} : borrower.fullName
```

### Debounced Search

Search input uses local state with debounced API calls to prevent excessive requests:

```typescript
const [localSearchQuery, setLocalSearchQuery] = useState('');

useEffect(() => {
  const timer = setTimeout(() => {
    if (localSearchQuery !== searchQuery) {
      setPage(0);
      fetchBorrowers(localSearchQuery, rowsPerPage, 0);
    }
  }, 300);

  return () => clearTimeout(timer);
}, [localSearchQuery]);
```

### Loading States

All list pages use MUI Skeleton components during loading, rendering the same number of skeleton rows as `rowsPerPage` for consistent height:

```typescript
{isLoading ? (
  Array.from({ length: rowsPerPage }).map((_, index) => (
    <TableRow key={index}>
      <TableCell><Skeleton /></TableCell>
      // ... more skeleton cells
    </TableRow>
  ))
) : (
  // ... actual data
)}
```

### Pagination

Server-side pagination with local state management:

```typescript
const handleChangePage = (_event: unknown, newPage: number) => {
  setPage(newPage);
  fetchDocuments(filters, rowsPerPage, newPage * rowsPerPage);
};
```

Offset calculation: `offset = page * rowsPerPage`

## API Integration

### Documents Endpoints

- **GET /api/documents** - List documents
  - Query params: `limit`, `offset`, `status`, `borrowerId`
  - Returns: `{ data: DocumentRecord[], pagination: PaginationInfo }`

- **GET /api/documents/:id** - Get single document
  - Returns: `DocumentRecord`

### Borrowers Endpoints

- **GET /api/borrowers** - List borrowers
  - Query params: `limit`, `offset`, `search`
  - Returns: `{ data: BorrowerRecord[], pagination: PaginationInfo }`

- **GET /api/borrowers/:id** - Get single borrower
  - Returns: `BorrowerRecord`
